# Техническое задание: поиск PDF-чертежей

## 1. Цель
Реализовать проверку наличия PDF-чертежей для деталей из таблицы с выбором алгоритма `ТИП 1..5` по префиксу детали и отображением результата в UI (missing/duplicate/success).

---

## 2. Входные данные

### 2.1 Данные строки таблицы
Для каждой строки используются поля:
- `Обозначение` — полное имя детали (основа для поиска PDF);
- `Наименование` — колонка для подсветки и вывода в отчёте;
- `Тип` — фильтр участия строки в PDF-проверке.

### 2.2 Настройки верификации
В настройках есть блоки `ТИП 1..5` с полем `Условие`.
По этим условиям определяется, какой алгоритм поиска PDF применять для конкретной детали.

### 2.3 Индекс файлов
Поиск выполняется по уже проиндексированным PDF-файлам (чертежам), полученным сервером через индексатор.

---

## 3. Какие строки проверяем
PDF-поиск выполняется только для строк, где `Тип` равен:
- `Деталь`
- `Деталь_кон`
- `Деталь_св`

Остальные строки не участвуют в PDF-проверке.

---

## 4. Как определить ТИП алгоритма (1..5)
Для каждой проверяемой строки:
1. Берём `Обозначение` (например: `StSwO.18.3x60.4.001-LC.BNC.V2`).
2. Выделяем префикс до первой точки `.` (в примере это `StSwO`).
3. Ищем этот префикс в условиях `ТИП 1..5`.
4. Если найдено соответствие:
   - совпало с `ТИП 1` → использовать алгоритм `Type1`;
   - совпало с `ТИП 2` → алгоритм `Type2`;
   - ...
   - совпало с `ТИП 5` → алгоритм `Type5`.
5. Если префикс не найден ни в одном из условий `ТИП 1..5`:
   - по умолчанию использовать алгоритм `Type1`.

---

## 5. Алгоритмы поиска

### 5.1 Type1 (детализированная версия)
Для деталей, попавших под `ТИП 1`, поиск выполняется так:
1. Взять полное `Обозначение`.
2. Проверить наличие символа `-`.
3. Если `-` есть — отбросить всё, что справа от первого `-` (включая сам `-`).
4. Если `-` нет — оставить `Обозначение` как есть.
5. Полученную строку использовать как базовое имя для поиска PDF-файла.
6. Считать успешным, если найден ровно один PDF с таким именем (без учёта регистра расширения).

Примеры:
- Деталь: `BNC.V2.0127.01.r1-LA` → искать файл `BNC.V2.0127.01.r1.PDF`.
- Деталь: `BNC.V2.0116.02` → искать файл `BNC.V2.0116.02.PDF`.
- Деталь: `St.00.003-L` → искать файл `St.00.003.PDF`.

### 5.2 Type2..Type5 (пока каркас)
Для `ТИП 2`, `ТИП 3`, `ТИП 4`, `ТИП 5` пока требуется оставить отдельные функции-каркасы:
- `ApplyType2Algorithm(...)`
- `ApplyType3Algorithm(...)`
- `ApplyType4Algorithm(...)`
- `ApplyType5Algorithm(...)`

Детальная логика для этих типов будет добавляться поэтапно.

---

## 6. Результат поиска и статусы
Для каждой проверяемой детали:
1. Найден ровно 1 файл → OK.
2. Не найдено ни одного файла → `missing`.
3. Найдено 2+ файла → `duplicate`.

---

## 7. Требования к UI

### 7.1 Подсветка в таблице
Подсветка выполняется в колонке `Наименование`:
- `missing` → светло-красный фон;
- `duplicate` → светло-голубой фон.

### 7.2 Диалог после верификации
После завершения проверки показывать новое диалоговое окно:
- список деталей, для которых PDF не найден;
- список дубликатов (имя детали + найденные пути файлов).

### 7.3 Успешный итог
Если все PDF найдены:
- вывести текст: **«Все файлы PDF найдены»**;
- стиль текста: **жирный** и **зелёный**.

---

## 8. API (минимум)
`POST /api/verification/kd` должен возвращать данные, достаточные для:
- подсветки строк по статусам `missing/duplicate`;
- формирования списка отсутствующих деталей;
- формирования списка дубликатов с путями.

---

## 9. Критерии приёмки
1. PDF-проверка выполняется только для `Тип = Деталь | Деталь_кон | Деталь_св`.
2. Для каждой детали корректно определяется префикс (до первой точки) и по нему выбирается `ТИП 1..5`.
3. Если префикс не попал ни в один `ТИП`, применяется `Type1` по умолчанию.
4. Для `ТИП 1` применяется правило удаления «мусора» после `-`.
5. Примеры отрабатывают корректно:
   - `BNC.V2.0127.01.r1-LA` → `BNC.V2.0127.01.r1.PDF`;
   - `BNC.V2.0116.02` → `BNC.V2.0116.02.PDF`;
   - `St.00.003-L` → `St.00.003.PDF`.
6. При `missing` подсветка светло-красная + деталь попадает в список «не найдено».
7. При `duplicate` подсветка светло-голубая + в диалоге выводятся пути дубликатов.
8. При полном успехе выводится сообщение **«Все файлы PDF найдены»** зелёным жирным текстом.
