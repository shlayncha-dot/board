# Техническое задание: корректировки интерфейса SLS Planning

## 1. Подсветка элементов при наведении мышью

### Цель
Обеспечить визуальную реакцию элементов интерфейса при наведении курсора.

### Требования
1. Для кнопок верхнего меню реализовать hover-подсветку:
   - подсветка рамки и тени у иконки раздела;
   - затемнение/акцент текста подписи пункта меню.
2. Для кнопок бокового подменю реализовать hover-состояние:
   - светлый фон без активации пункта.
3. Для переключателей языка реализовать hover-состояние:
   - легкое выделение фона кнопки.

### Критерий приемки
При наведении курсора мыши на соответствующие элементы пользователь видит подсветку до клика.

---

## 2. Разделение настроек «Аккаунта» и «Модулей»

### Цель
Исключить пересечение экранов настроек из разных контекстов.

### Требования
1. Настройки аккаунта должны открываться **только** из пользовательского меню в шапке.
2. Пункты «Настройки/Настройка» в модульных подменю должны работать как обычные пункты текущего модуля и не открывать форму аккаунта.
3. Состояние экрана «Настройки аккаунта» хранить отдельно от состояния активного пункта подменю.

### Критерий приемки
- Клик по «Настройки» в модульном меню не открывает профильные настройки.
- Клик по «Настройки аккаунта» в пользовательском меню открывает только экран профиля.

---

## 3. Поведение при выходе из системы

### Цель
После выхода оставлять на экране только информационную верхнюю панель.

### Требования
1. При нажатии «Выйти» скрывать:
   - верхнюю иконковую навигацию;
   - рабочую область и боковое меню;
   - пользовательский блок аккаунта в шапке.
2. Оставлять видимой информационную шапку с логотипом, датой/временем, названием системы и языковым переключателем.

### Критерий приемки
После выхода отображается только верхняя информационная панель и окно авторизации.

---

## 4. Новый экран авторизации

### Цель
Заменить выбор пользователя по аватару на стандартную форму входа.

### Требования
1. Экран входа должен содержать:
   - поле «Логин»;
   - поле «Пароль»;
   - чекбокс «Запомнить меня»;
   - кнопку «Войти».
2. Запрещено использовать аватары/кружки/карточки выбора пользователя.
3. Для совместимости с автозаполнением браузера задать корректные атрибуты:
   - `autocomplete="username"` для логина;
   - `autocomplete="current-password"` для пароля.
4. При отмеченном чекбоксе сохранять логин локально для подстановки при следующем входе.

### Критерий приемки
Пользователь видит только форму логин/пароль + чекбокс. Браузер может предложить сохранить учетные данные в менеджере паролей.

---

## 5. Нефункциональные требования

1. Изменения не должны ломать существующую навигацию по разделам.
2. Код должен соответствовать действующим правилам линтинга проекта.
3. Визуальные изменения должны быть реализованы средствами CSS без сторонних библиотек.

## 6. Проверка и тестирование

1. Линтинг фронтенд-проекта проходит без ошибок.
2. Сборка фронтенд-проекта проходит успешно.
3. Проверены сценарии вручную:
   - hover-подсветка всех требуемых кнопок;
   - независимость настроек аккаунта и модулей;
   - выход из системы с сохранением только верхней панели;
   - корректная работа формы логина и чекбокса «Запомнить меня».

---

## 7. Экран «Настройки пользователей» (админ)

### Цель
Заменить текущую форму настройки пользователя на табличный режим редактирования.

### Требования
1. В пункте «Настройки пользователей» отображать только таблицу пользователей и кнопки действий.
2. Колонки таблицы (в порядке слева направо):
   - № (порядковый номер);
   - Логин;
   - Имя;
   - Фамилия;
   - Роль;
   - Статус админ;
   - Телефон;
   - Емейл;
   - Фото;
   - Пароль.
3. Колонки «Роль» и «Статус админ» должны быть редактируемыми через выпадающие списки.
4. Внизу экрана должны быть кнопки:
   - «Сохранить» — применяет изменения ролей/админ-статуса;
   - «Отмена» — сбрасывает несохранённые изменения в таблице.
5. Стиль таблицы должен соответствовать стилю экрана «Проверка спецификации»: чёрная/тёмная пунктирная рамка и разделители.

### Критерий приемки
- В меню «Настройки пользователей» отсутствует старая форма выбора одного пользователя.
- Таблица отображает весь список пользователей по строкам.
- Роль и админ-статус меняются через выпадающие списки.
- Кнопки «Сохранить» и «Отмена» работают по требованиям.
