# ТЗ: Windows-батник для индексатора PDF-файлов

## 1. Цель
Сделать «полевой» запуск индексатора на удалённом Windows-ПК максимально простым: пользователь запускает один `.bat`, скрипт регулярно сканирует локальную папку, индексирует **все файлы** (имя + путь + метаданные) и отправляет снимок на сервер `SLS-Planning` через HTTP API.

## 2. Что хотели получить
- Один вход для пользователя: `run-indexer.bat`.
- Без установки IDE/сборки: только батник + PowerShell-скрипт + `config.json`.
- Стабильная фоновая работа (опционально через Планировщик задач).
- Диагностируемость: понятные логи в консоли при старте, сканировании и отправке.
- Без прямого доступа к БД: только вызов API `/api/file-index/sync`.

## 3. Состав поставки
- `tools/windows-indexer/run-indexer.bat` — точка запуска.
- `tools/windows-indexer/indexer.ps1` — логика сканирования и синка.
- `tools/windows-indexer/config.json.example` — шаблон конфигурации.
- `tools/windows-indexer/install-autostart-task.bat` — автозапуск через Task Scheduler.

## 4. Функциональные требования
1. Скрипт читает конфиг `config.json` рядом с батником.
2. Валидирует обязательные поля: `serverUrl`, `syncEndpoint`, `scanRoot`, `scanIntervalSeconds`, `machineId`.
3. Проводит периодическое сканирование `scanRoot` и формирует список всех файлов.
4. Отправляет результаты на `POST {serverUrl}{syncEndpoint}`.
5. Поддерживает ограничение размера запроса (`maxPayloadBytes`) и отправку чанками.
6. Поддерживает таймаут запроса (`requestTimeoutSeconds`) и ретраи при временных сбоях.
7. Поддерживает опциональную HTTP-авторизацию (`auth`), если её требует прокси/сервер.
8. Пишет этапы в лог: старт, количество файлов, отправка, ошибки, повторные попытки.

## 5. Нефункциональные требования
- Запуск на стандартном Windows с PowerShell 5.1+.
- Работает без admin-прав для ручного запуска (admin нужен только для установки автозапуска).
- Корректная работа при больших деревьях файлов.
- Устойчивость к сетевым ошибкам и недоступности сервера.

## 6. UX и эксплуатация
- Пользовательский сценарий:
  1) скопировать папку `windows-indexer`;
  2) создать `config.json` по примеру;
  3) запустить `run-indexer.bat`.
- Для постоянной работы — однократно выполнить `install-autostart-task.bat` от администратора.

## 7. Критерии приёмки
- Батник стартует индексатор без ручного ввода параметров.
- В логах видно: `Indexer started`, путь сканирования, интервал, таймауты, число найденных файлов.
- При большом объёме данных отправка не падает по размеру одного запроса (используются чанки).
- При недоступности API есть повторные попытки и понятная ошибка.
- После восстановления сети синхронизация продолжается.

## 8. Что отдельно зафиксировали по инцидентам
- Ошибки вида `ParserError`, `MissingEndParenthesisInMethodCall` обычно означают повреждённую/старую копию `indexer.ps1`.
- Базовый процесс восстановления: обновить папку `windows-indexer` из репозитория и запускать именно `run-indexer.bat` из этой папки.

## 9. Границы задачи
- Батник/скрипт **не** подключается напрямую к базе данных.
- Задача индексатора — передать файловый снимок на сервер; хранение и последующая логика поиска — зона backend.
